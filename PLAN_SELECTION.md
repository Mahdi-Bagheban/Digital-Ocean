## ⚠️ نكات مهم در انتخاب

### **• Latency (تأخیر)**
منطقه را بر اساس نزدیكم به كاربران خود انتخاب کنید:

- **اروپا**: `fra1` (Frankfurt) 或 `ams3` (Amsterdam)
- **آمریکا**: `nyc1` (New York) یا `sfo3` (San Francisco)
- **آسیا**: `sgp1` (Singapore)

### **• Bandwidth**

Space از عد بیرون: (Outbound Transfer)
- پلن‌های بزرگ: 8-12 TB/ماه
- ارزان‌ها: 1-2 TB/ماه

### **• Database Size**
اگر DBخود دارید:
- RAM هر 10 GB per core الزم است
- `m-16vcpu-64gb` برای DBهای خلوت نامه (< 50GB) چک است

### **• Backup Strategy**

```
Enable Backups = true?
✓ اگر دادهه مهم دارهاش باشد
✓ یا بابت جدید و Experimental
✗ اگر فقط حیطه راه خود خالی غرقہ کنید
✗ برای کاهش هزینه‌ها
```

## 📊 راهنمای Scale Up/Down

### **Scale Up (بزرگ‌تر کردن)**

اگر دیدید سرور شما کوتاه است:

1. سرور فعلی را حذف کنید (یا Snapshot بگیرید)
2. برای پلن بزرگتر Workflow اجرا کنید
3. روز بعدی، اطلاعات میروق از Snapshot بازیابی کنید

### **Scale Down (کوتاه‌تر کردن)**

اگر نمي‌خواهيد هزینه کم كنيد:

1. سرور فعلی را حذف کنید
2. پلن کوچک‌تر انتخاب کنید (`s-2vcpu-4gb`)
3. اجرا Workflow
